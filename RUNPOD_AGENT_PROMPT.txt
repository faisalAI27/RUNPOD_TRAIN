I am working on a deep learning project for ASL (American Sign Language) sign language detection using Faster R-CNN with PyTorch. I need you to set up the environment, train the model, and — most importantly — back up the trained checkpoint to GitHub so it is never lost when the pod shuts down. Here is the complete situation:

Project repo: https://github.com/faisalAI27/RUNPOD_TRAIN.git

What the repo contains:
- src/ folder with all training code: config.py, dataset.py, transforms.py, dataloaders.py, model.py, train.py, utils.py, visualize.py
- data/ folder — currently empty, needs to be filled with images
- outputs/checkpoints/ — where the trained model will be saved as frcnn_best.pth
- prepare_kaggle_data.py — script to download and prepare data from Kaggle if needed

About the dataset:
The dataset is the Kaggle ASL Alphabet dataset (grassknoted/asl-alphabet). It has 13,000 images split into data/train/ (9,100 images), data/valid/ (1,950 images), and data/test/ (1,950 images). Each image has a corresponding Pascal VOC XML annotation file with bounding box coordinates. All 26 letters A-Z are included as classes.

About the model:
Faster R-CNN with ResNet-50 + FPN backbone, pretrained on COCO, fine-tuned for 27 classes (26 letters + background). Training runs for 40 epochs using SGD with learning rate 0.005, StepLR scheduler (step=20, gamma=0.1), batch size 4. The best checkpoint (lowest validation loss) is saved automatically.
